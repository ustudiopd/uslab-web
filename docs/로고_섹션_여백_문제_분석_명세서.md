# 로고 섹션 여백 문제 분석 명세서

**작성일**: 2025-12-14  
**문제**: Hero 섹션의 로고 이미지와 다음 섹션(Philosophy) 사이의 여백이 과도하게 큼

---

## 1. 문제 현상

### 1.1 사용자 요구사항
- **목표**: 이전 버전처럼 로고 섹션 주변의 여백을 최소화
- **현재 상태**: 로고 아래에 과도한 여백이 존재하여 "붕 떠있는" 느낌
- **참고 이미지**: 이전 버전에서는 border 위아래로 간격이 매우 적었음

### 1.2 시도한 해결 방법
1. ✅ 로고 섹션의 `mt` (상단 여백) 조정: `mt-2 sm:mt-3`
2. ✅ 로고 섹션의 `pt` (패딩) 조정: `pt-1 sm:pt-2`
3. ✅ Hero 섹션의 `pb` (하단 여백) 조정: `pb-4 sm:pb-6 lg:pb-8`
4. ❌ **결과**: 여전히 간격이 크게 유지됨

---

## 2. 근본 원인 분석

### 2.1 여백이 누적되는 구조

```
[Hero Section]
  └─ pb-4 sm:pb-6 lg:pb-8 (하단 여백)
     └─ [로고 섹션]
        └─ mt-2 sm:mt-3 (상단 여백)
        └─ pt-1 sm:pt-2 (패딩)
        └─ [로고 이미지]
  └─ pb-4 sm:pb-6 lg:pb-8 (Hero 섹션 하단 여백) ← 여기서 여백 발생

[Philosophy Section]
  └─ py-12 sm:py-16 lg:py-24 (상하 여백)
     └─ pt-12 sm:pt-16 lg:pt-24 (상단 여백) ← 여기서도 여백 발생
```

### 2.2 실제 여백 계산

**데스크톱 (lg) 기준:**
- Hero 섹션 하단 여백: `pb-8` = 2rem (32px)
- Philosophy 섹션 상단 여백: `pt-24` = 6rem (96px)
- **총 여백**: 32px + 96px = **128px** ← 이게 문제!

**태블릿 (sm) 기준:**
- Hero 섹션 하단 여백: `pb-6` = 1.5rem (24px)
- Philosophy 섹션 상단 여백: `pt-16` = 4rem (64px)
- **총 여백**: 24px + 64px = **88px**

**모바일 기준:**
- Hero 섹션 하단 여백: `pb-4` = 1rem (16px)
- Philosophy 섹션 상단 여백: `pt-12` = 3rem (48px)
- **총 여백**: 16px + 48px = **64px**

### 2.3 코드 현황

#### Hero.tsx (현재)
```tsx
<section className="relative pt-24 pb-4 sm:pt-32 sm:pb-6 lg:pt-52 lg:pb-8">
  {/* ... */}
  <div className="mt-2 sm:mt-3 border-t border-slate-800 pt-1 sm:pt-2">
    {/* 로고 이미지 */}
  </div>
</section>
```

#### Philosophy.tsx (현재)
```tsx
<section className="py-12 sm:py-16 lg:py-24 bg-slate-950 relative border-t border-slate-900">
  {/* ... */}
</section>
```

## 3. 왜 수정이 반영되지 않는가?

### 3.1 문제점 1: 섹션 간 여백이 누적됨
- Hero 섹션의 `pb`와 Philosophy 섹션의 `pt`가 **동시에 적용**되어 여백이 두 배로 누적됨
- 로고 섹션 내부의 여백만 줄여서는 해결되지 않음

### 3.2 문제점 2: Philosophy 섹션의 상단 여백이 고정됨
- Philosophy 섹션은 `py-12 sm:py-16 lg:py-24`로 **상하 여백이 동일**하게 설정됨
- 상단 여백만 줄이면 하단 여백도 함께 줄어드는 부작용 발생

### 3.3 문제점 3: 로고 섹션이 Hero 섹션 내부에 있음
- 로고 섹션을 Hero 섹션 밖으로 분리하지 않으면 Hero의 `pb`가 항상 적용됨
- 로고 섹션만을 위한 독립적인 여백 제어가 어려움

---

## 4. 해결 방안

### 방안 1: Philosophy 섹션의 상단 여백만 선택적으로 줄이기 (권장)

**장점**: 
- Hero 섹션 구조 유지
- Philosophy 섹션의 하단 여백은 유지

**구현:**
```tsx
// Philosophy.tsx
<section className="pt-4 sm:pt-6 lg:pt-8 pb-12 sm:pb-16 lg:pb-24 bg-slate-950 relative border-t border-slate-900">
  {/* 상단 여백만 줄이고, 하단 여백은 유지 */}
</section>
```

**예상 결과:**
- 데스크톱: 32px (Hero pb) + 32px (Philosophy pt) = **64px** (현재 128px에서 절반)
- 태블릿: 24px + 24px = **48px** (현재 88px에서 절반)
- 모바일: 16px + 16px = **32px** (현재 64px에서 절반)

### 방안 2: Hero 섹션의 하단 여백을 0으로 설정

**장점**: 
- 로고 바로 아래 여백 제거

**단점**: 
- Hero 섹션의 다른 요소들도 영향을 받을 수 있음

**구현:**
```tsx
// Hero.tsx
<section className="relative pt-24 pb-0 sm:pt-32 sm:pb-0 lg:pt-52 lg:pb-0">
  {/* ... */}
</section>
```

### 방안 3: 로고 섹션을 Hero 밖으로 분리

**장점**: 
- 로고 섹션만 독립적으로 여백 제어 가능

**단점**: 
- 구조 변경이 큼
- Hero 섹션의 배경 효과와 분리될 수 있음

**구현:**
```tsx
// app/[lang]/page.tsx
<Hero />
<LogoSection /> {/* 새 컴포넌트 */}
<Philosophy />
```

### 방안 4: 음수 마진 사용 (임시 방편)

**장점**: 
- 빠른 적용 가능

**단점**: 
- 레이아웃이 깨질 수 있음
- 반응형 대응이 어려움

**구현:**
```tsx
// Hero.tsx
<div className="mt-2 sm:mt-3 border-t border-slate-800 pt-1 sm:pt-2 flex justify-center -mb-8 sm:-mb-12 lg:-mb-16">
  {/* 음수 마진으로 Philosophy 섹션과 겹치게 */}
</div>
```

## 5. 권장 해결책

### 최종 권장: 방안 1 (Philosophy 섹션 상단 여백만 줄이기)

**이유:**
1. Hero 섹션 구조를 유지하면서 문제 해결 가능
2. Philosophy 섹션의 하단 여백은 유지하여 다른 섹션과의 간격 보장
3. 가장 안전하고 예측 가능한 방법

**구현 코드:**
```tsx
// components/sections/Philosophy.tsx
<section
  id="about"
  className="pt-4 sm:pt-6 lg:pt-8 pb-12 sm:pb-16 lg:pb-24 bg-slate-950 relative border-t border-slate-900"
>
  {/* py-12 sm:py-16 lg:py-24 대신 pt와 pb를 분리 */}
</section>
```

**추가 조정 (필요시):**
```tsx
// Hero.tsx - 로고 섹션 여백도 더 줄이기
<div className="mt-1 sm:mt-2 border-t border-slate-800 pt-0.5 sm:pt-1 flex justify-center">
```

## 7. 참고사항

- Tailwind CSS의 `py-*` 클래스는 `pt-*`와 `pb-*`를 동시에 설정
- 섹션 간 여백은 두 섹션의 여백이 **합산**됨
- 로고 섹션 내부 여백만 조정해서는 섹션 간 여백 문제 해결 불가
- **Philosophy 섹션의 상단 여백을 줄여야** 로고와 Philosophy 사이 간격이 줄어듦
