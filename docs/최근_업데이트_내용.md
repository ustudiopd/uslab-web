**작성일**: 2025-01-25

## 1. 히트맵 기능 고도화 (v2)

### 주요 개선사항
- **정확한 위치 추적**: 긴 페이지에서도 스크롤 위치를 고려한 정확한 클릭 위치 추적
- **고해상도 히트맵**: 20×20 그리드에서 200×200 그리드로 해상도 향상 (10배 증가)
- **부드러운 Blob 렌더링**: 기존 격자 형태에서 부드럽게 번지는 히트맵으로 시각적 품질 향상
- **실시간 스크롤 반응**: 스크롤 시 딜레이 없이 즉시 히트맵이 따라붙도록 최적화

### 사용 방법
- 대시보드 → "메인페이지 히트맵 보기" 버튼 클릭
- 필터 옵션: 기간(7일/30일), 디바이스(전체/모바일/데스크톱), 해상도(100×100/200×200/300×300)
- 렌더링 설정: 반경, 블러, 투명도 조절 가능

## 2. 페이지 성능 개선 (CLS 최적화)

### 개선 내용
- **이미지 로딩 최적화**: 이미지 크기 정보를 미리 설정하여 레이아웃 시프트 방지
- **YouTube 영상 최적화**: 초기 크기를 명시하여 로딩 전 공간 확보
- **폰트 로딩 최적화**: 폰트 로딩 지연 시에도 텍스트 크기 변경 최소화

### 효과
- 페이지 로딩 시 레이아웃이 흔들리는 현상 감소
- 사용자 경험 향상

---

## 3. 블로그 포스트 가독성 개선

### 개선 내용
- **코드 블록 스타일 개선**: 라이트 테마에서 코드 블록의 배경색과 텍스트 색상 대비 향상
- **줄별 하이라이트 제거**: 코드 블록 내부의 불필요한 배경색 제거로 가독성 향상
- **복사 버튼 스타일 개선**: 라이트 테마에서 복사 버튼의 가독성 향상

---

## 4. 대시보드 개선

### 개선 내용
- **관리자 페이지 제외**: "인기 클릭 요소"와 "페이지별 클릭"에서 `/admin` 경로 제외
- 공개 페이지 데이터만 표시하여 더 유의미한 분석 가능

---

## 기술적 세부사항

### 히트맵 v2 주요 기술
- 문서 기준 좌표 시스템 (`page_x`, `page_y`)
- 뷰포트 고정 캔버스 (메모리 절약)
- Blur 누적 + Colorize 알고리즘
- 외부 라이브러리 없이 순수 Canvas API 사용

### 성능 최적화
- 스크롤 이벤트 즉시 처리 (throttle 제거)
- 화면 밖 포인트 자동 제외
- 샘플링 전략으로 대용량 데이터 처리

---

**참고**: 모든 변경사항은 하위 호환성을 유지하며 기존 기능에 영향을 주지 않습니다.
